## 📋 概述

本文档定义了项目中 Agent 的开发规范和技术标准，确保代码质量和团队协作效率。

<IMPORTANT>
## 重要准则
- 严格按照当前项目的开发组织习惯。请严格阅读mfg-process/README.md下的`应用目录结构说明和规范`，可微调，总体按照这个组织方式。注意优先封装，控制文件代码行数，禁止过于复杂的代码和安全检查。
- 禁止随意回滚用户的操作，只需要顾好自己的改动即可。 用户可能在你改动时边修改其他功能点。除非与当前正在修改的功能点明确冲突，否则禁止向用户询问，若不冲突则直接忽视。
- 给出多步执行命令的时候，尽量用# 1) xxx标题 \n ${执行的命令} # 2) xxx标题 \n ${执行的命令}
- 严格禁止兼容旧代码，每次以完全重构的标准来执行。
- 系统级的更新记得同步到README.md或者相关的记录文档里。同步时先看有没有相关记录，有则修改，否则新增。
- 当我的提问里包含ulw的时候，严令禁止再向用户提出任何确认/问题，此时碰到问题应深度思考，参考行业标准多维度评估后选择合适推荐的选项。如果问题涉及到资源，则应考虑到用户资源受限，以免费/低成本为先。
- 每轮plan任务结束执行以后1以命令行形式给出以git add 需要add的文件（换行符连接文件, 禁止添加被ignore的文件）及本轮任务的推荐commit信息。
- 连续的命令行如果存在注释比如1) 2)前面需要加#表示只是注释内容。
</IMPORTANT>
